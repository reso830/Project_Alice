# Implementation Plan: Profile Page Refinements

**Branch**: `009-profile-page-refinement` | **Date**: 2026-04-29 | **Spec**: [spec.md](spec.md)

## Summary

Refine the Profile page and Edit Profile page in three parallel tracks:

1. **View Profile visual alignment** — refactor `renderCertifications` and `renderAwards` in `src/pages/Profile.js` to use the same `profile-entry-list` / `profile-entry` DOM structure currently used by Education and Professional Experience. Certifications gain a title/meta/ID hierarchy; Awards gain a title/meta/description hierarchy.

2. **Edit Profile interaction model rewrite** — in `src/pages/ProfileEdit.js`, (a) reorder section cards to match the View Profile order, (b) upgrade `createEntryRow` into a `createStructuredEntryRow` that mirrors the View Profile hierarchy and adds Edit + Remove icon buttons, (c) move the Add button from the card body into the section card header with primary styling and the label "Add", (d) replace every inline add form with a `createEntryOverlay` function that renders a modal on desktop (≥ 640 px) or a bottom sheet on mobile (< 640 px), (e) wire Edit icon buttons to open the same overlay pre-filled with existing entry data.

3. **iPad Mini stat chip fix** — a single CSS rule scoped to `.apps-desktop-vis__stats .stat-chip-row` switches the chip grid from `repeat(4, minmax(0, 1fr))` to `repeat(2, minmax(0, 1fr))` so chips wrap into a 2 × 2 block at mid-range viewport widths without affecting the desktop side-column layout or the mobile row layout.

No new npm dependencies are added. No backend changes are required. No data model changes are required.

### 4. Edit Profile — Structured entry row with Edit + Remove icons

**Replace** the current flat `createEntryRow(parts, onRemove)` with a new `createStructuredEntryRow(display, { onEdit, onRemove })`:

```js
function createStructuredEntryRow(display, { onEdit, onRemove } = {}) {
  const row = createElement('div', 'entry-row entry-row--structured');
  const content = createElement('div', 'entry-row__content');
  const actions = createElement('div', 'entry-row__actions');

  if (display.title) content.append(createElement('div', 'profile-entry__title', display.title));
  if (display.meta)  content.append(createElement('div', 'profile-entry__meta',  display.meta));
  if (display.desc)  content.append(createElement('p',   'profile-entry__desc',  display.desc));

  if (onEdit) {
    actions.append(createButton('✎', 'entry-row__edit',   onEdit,   'Edit entry'));
  }
  actions.append(createButton('×', 'entry-row__remove', onRemove, 'Remove entry'));

  row.append(content, actions);
  return row;
}
```

Each section card calls `createStructuredEntryRow` for its entries, providing both `onEdit` and `onRemove` callbacks (except Skills, which keeps pill-based display).

### 6. Entry Overlay — Modal / Bottom Sheet system

A single `createEntryOverlay(title, buildForm, { onSave, initialValues })` function creates the appropriate container based on viewport width. This replaces all seven inline form patterns.

```js
function createEntryOverlay(title, buildForm, { onSave, initialValues = {} } = {}) {
  const isDesktop = window.innerWidth >= 640;
  const overlay = createElement('div', isDesktop ? 'entry-modal' : 'entry-sheet');
  const backdrop = createElement('div', 'entry-overlay-backdrop');
  const container = createElement('div', isDesktop ? 'entry-modal__box' : 'entry-sheet__box');
  const header = createElement('div', 'entry-overlay__header');
  const titleEl = createElement('h2', 'entry-overlay__title', title);
  const formEl = createElement('div', 'entry-overlay__form');
  const footer = createElement('div', 'entry-overlay__footer');

  let _formFields = {};          // populated by buildForm
  let _isDirtyFn = () => false;  // set by buildForm; returns true if form changed

  const { fields, isDirty: formIsDirty } = buildForm(formEl);
  _formFields = fields;
  _isDirtyFn = formIsDirty;

  const cancelBtn = createButton('Cancel', 'profile-btn profile-btn--outline', handleCancel);
  const saveBtn   = createButton('Save',   'profile-btn profile-btn--primary',  handleSave);

  footer.append(cancelBtn, saveBtn);
  header.append(titleEl);
  container.append(header, formEl, footer);
  overlay.append(container);
  document.body.append(backdrop, overlay);
  document.body.style.overflow = 'hidden';

  // Focus trap: Tab cycles within overlay
  overlay.addEventListener('keydown', handleOverlayKeydown);
  backdrop.addEventListener('click', handleCancel);

  function handleSave() {
    if (!validateOverlayForm(_formFields)) return;
    const data = collectOverlayData(_formFields);
    onSave(data);
    close();
  }

  function handleCancel() {
    if (!_isDirtyFn()) { close(); return; }
    showOverlayDiscardDialog(container, { onDiscard: () => { Toast.show('Changes discarded.', 'success'); close(); } });
  }

  function close() {
    document.body.style.overflow = '';
    overlay.remove();
    backdrop.remove();
  }

  // Return handle so external code (e.g., open inline error) can close if needed
  return { close };
}
```

**Form builder pattern**: Each section card passes a `buildForm(formEl)` function that appends the section's fields into `formEl` using `createField` / `createSelectField`, then returns `{ fields, isDirty }`. The `isDirty` function serializes current field values and compares to the `initialValues` snapshot passed to `createEntryOverlay`.

```js
// Example: Experience section
function buildExperienceForm(formEl, initial = {}) {
  const role    = createField('Role',            initial.role    ?? '', false, { required: true });
  const company = createField('Company',         initial.company ?? '', false, { required: true });
  // ... other fields
  const dateRow = createElement('div', 'inline-entry-form__row inline-entry-form__row--dates');
  dateRow.append(dateStarted.wrapper, dateEnded.wrapper, currentWorkWrapper);
  formEl.append(role.wrapper, company.wrapper, responsibilities.wrapper, dateRow);

  const fields = { role, company, responsibilities, dateStarted, dateEnded, currentWork };
  const snapshot = JSON.stringify(getFormData(fields));
  const isDirty = () => JSON.stringify(getFormData(fields)) !== snapshot;

  return { fields, isDirty };
}
```

**isDirty snapshot note**: The snapshot is taken from `getFormData(fields)` *after* field initialization — not from `initialValues` directly. This prevents false-positive dirty detection in add mode, where `initialValues = {}` but `getFormData` returns `{ role: '', company: '', ... }`. Comparing `{}` to `{ role: '' }` would always yield "dirty"; snapshotting the initialized form avoids this.

**Keyboard focus trap**: ESC key triggers `handleCancel()`. Tab key is constrained to focusable elements inside the overlay.

### 7. Per-section modal wiring

Each section card is refactored to:

1. Call `createEditCard(title, { onAdd: openAddOverlay })` — Add button goes in header.
2. Replace `createEntryRow(...)` with `createStructuredEntryRow(display, { onEdit, onRemove })` for its entries.
3. `openAddOverlay`: calls `createEntryOverlay(title, buildSectionForm, { onSave: (data) => { _formState.section.push(data); commitListChange(); render(); } })`.
4. `onEdit` callback (per entry): calls `createEntryOverlay(title, (el) => buildSectionForm(el, existingEntry), { onSave: (data) => { _formState.section.splice(index, 1, data); ...; render(); } })`.

Experience and Education re-apply `sortExperience` / `sortEducation` inside their `onSave` callback before calling `render()`.

**Sort reference identity note**: `sortExperience` and `sortEducation` MUST return the same object references — not clones. The `indexOf`-based edit identity (`_formState.experience.indexOf(entry)`) depends on reference equality. The current sort implementation reorders array elements in place (or via reference-preserving comparisons) and is safe. Do not refactor these functions to return new object copies.

**Open overlay guard**: Store a `_openOverlay` module variable. When `createEntryOverlay` is called, if `_openOverlay` is already set, return early (Add button click has no effect). Clear `_openOverlay` in `close()`.

### 8. Skills overlay — staged flow

Skills replaces its current inline pill/input pattern with a modal that maintains its own staged array:

```js
function openSkillsOverlay() {
  if (_openOverlay) return;
  const staged = [];  // modal-local staged skills (separate from _formState.skills)

  const overlay = createEntryOverlay('Add Skills', (formEl) => {
    // input + Add button + pill area (staged only)
    // staged pills mutate `staged[]` only; formEl is re-rendered each add/remove

    const isDirty = () => staged.length > 0;
    return { fields: {}, isDirty };
  }, {
    onSave: () => {
      // merge staged into _formState.skills, deduplicating case-insensitively
      const existing = new Set(_formState.skills.map((s) => s.toLowerCase()));
      for (const skill of staged) {
        if (!existing.has(skill.toLowerCase())) {
          _formState.skills.push(skill);
          existing.add(skill.toLowerCase());
        }
      }
      commitListChange();
      renderSkillsCard.render();  // re-render the skills section body
    },
  });
  _openOverlay = overlay;
}
```

The inline skill text input + Add button inside the overlay adds to `staged[]` and re-renders the pill list inside the overlay only. Existing skills remain in `_formState.skills` untouched until Save is clicked.

Cancel with `staged.length > 0` triggers the overlay discard flow.

### 9. Overlay discard confirmation

The overlay's discard flow uses a lightweight second-layer dialog (not the global page-level discard modal) so both the page-level guard and overlay-level guard can coexist independently:

```js
function showOverlayDiscardDialog(boxEl, { onDiscard }) {
  const dialog = createElement('div', 'overlay-discard-dialog');
  const msg    = createElement('p',   'overlay-discard-dialog__msg', 'Discard changes to this entry?');
  const discard = createButton('Discard',      'profile-btn profile-btn--danger',   () => { dialog.remove(); onDiscard(); });
  const keep    = createButton('Keep Editing', 'profile-btn profile-btn--outline',  () => dialog.remove());

  dialog.append(msg, discard, keep);
  boxEl.append(dialog);
}
```

The dialog sits inside the open modal/sheet box, above the form, so the backdrop already provides the lock. No additional scroll-lock needed.

### 11. iPad Mini stat chip fix

The `apps-desktop-vis__stats` column houses a `stat-chip-row` with `repeat(4, minmax(0, 1fr))`. At ~768 px wide, the stats column is narrow enough that "Pending" overflows.

**Fix**: Add a scoped rule so the desktop stats column always shows chips in a 2 × 2 grid:

```css
.apps-desktop-vis__stats .stat-chip-row {
  grid-template-columns: repeat(2, minmax(0, 1fr));
}
```

This rule is outside any media query, so it applies at all desktop widths (≥ 640 px where `apps-desktop-vis` is visible). At wider viewports the 2 × 2 layout remains visually balanced since each chip is generously sized. **This 2 × 2 layout is intentional at all desktop widths — no upper-bound media query is needed.** The mobile stat chip row (inside `apps-mobile-vis`) uses its own existing responsive rules and is unaffected.

No changes to the mobile layout.
