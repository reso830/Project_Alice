### [X] Task 03.1 — Create `src/components/Timeline.js` with collapsed-only render

**Target file**: [src/components/Timeline.js](../../src/components/Timeline.js) (NEW)

**What to do**:
Export a `Timeline` module shaped like `StatusDropdown` (module-level
singleton with three exports). Initial scope:

```js
// module-level state — survives Modal._renderBody() re-instantiation
let _host = null;
let _draft = null;
let _expanded = false;
let _onChange = null;
let _currentStatus = null;

export function render(draft, { currentStatus, onChange }) {
  _draft = draft; _currentStatus = currentStatus; _onChange = onChange;
  const wrapper = /* build DOM */;
  _host = wrapper;
  return wrapper;
}

export function refresh() {
  // Used by Modal after a status-change auto-entry append. Rebuild
  // the contents of _host in-place from current _draft / _expanded.
  if (!_host) return;
  _host.replaceChildren(/* rebuild */);
}

export function reset() {
  // Called by Modal.close(). Clear module state so the next open
  // starts collapsed (FR-009 session-local invariant).
  _expanded = false;
  _host = null;
  _draft = null;
  _onChange = null;
  _currentStatus = null;
}

export const Timeline = { render, refresh, reset };
```

The `render` function:

1. Builds a wrapper `<div class="modal-field modal-field--full">`
   with label "Timeline" (using the same `.modal-field__label`
   class as the existing field labels).
2. Renders a collapsed-state row inside (`.tl-collapsed`) when
   `_expanded === false`. (Expanded DOM lands in Phase 04.)
3. The collapsed row shows:
   - If `sortTimelineEntries(draft.timeline)[0]` exists: chevron +
     date (`Mon D` via `toDisplayDate` from `utils/date.js`) +
     `.tl-dash` em-dash + status badge (reuses `createStatusBadge`
     from `utils/dom.js`) + note text (truncated single line).
   - Otherwise: italic empty-state line "No entries yet — click to
     add".
4. The collapsed row has `role="button"`, `tabIndex="0"`, and a
   focus-visible outline matching `--indigo`.
5. Clicking, pressing Enter, or pressing Space flips `_expanded`
   and calls `refresh()`. (In Phase 03 the expanded branch is a
   placeholder; in Phase 04 it builds the full expanded DOM.)

Do NOT introduce expanded-mode DOM in this task.

**Why `_expanded` is module-level**: `Modal._renderBody()` is called
after Save without closing the modal ([src/components/Modal.js:682, 722](../../src/components/Modal.js#L682)),
which replaces the body's children and would otherwise destroy
component state. A module-level boolean survives the re-instantiation
so the user's expanded view persists across Save. The `reset()`
export, called from `Modal.close()` (Task 03.2), flips it back to
`false` so reopening the overlay still starts collapsed per spec
FR-009. See [research.md R-9a / R-9b](../../specs/025-application-timeline/research.md).

**Expected behavior**:
- A rich-timeline application's Timeline row shows the newest
  entry single-line.
- An empty-timeline application shows the empty-state prompt.
- Pressing Enter / Space on the row fires the (placeholder)
  expand toggle.

**Constraints**:
- No localStorage / sessionStorage writes.
- No persisting `expanded` state across overlay opens — module-local
  is fine (Modal will re-instantiate on each open).
- Do not call `onChange` in this task — read-only render only.
- Do not import from `server/`.

**Validation**:

  text; asserts the row is focusable and has `role="button"`.

import { allocateTimelineEntryId } from '../models/application.js';
