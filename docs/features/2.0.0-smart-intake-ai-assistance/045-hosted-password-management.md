# Feature 045 — Hosted Password Management

## Status

Proposed

---

# Summary

Introduce comprehensive password management capabilities for Hosted Mode users, allowing authenticated users to change their password and unauthenticated users to securely recover access to their account through a self-service password reset workflow.

This feature applies exclusively to Hosted Mode and shall have no impact on Local Mode deployments.

---

# Problem Statement

Hosted users currently rely on administrative intervention to recover forgotten passwords and cannot manage their own credentials from within Alice.

Modern authentication systems are expected to provide secure self-service password management while protecting user privacy and preventing information disclosure.

---

# Goals

- Allow authenticated users to change their password.
- Allow unauthenticated users to recover forgotten passwords.
- Reduce operator involvement in password management.
- Leverage the Hosted Authentication provider's recovery capabilities.
- Maintain complete separation between Hosted Mode and Local Mode.

---

# Non-Goals

This feature does not include:

- Email address changes
- Username changes
- Multi-factor authentication (MFA)
- Passwordless authentication
- Account deletion
- Authentication provider migration
- Password strength meters beyond Alice's existing password policy

---

# User Stories

### US-1 — Change Password

As a Hosted Mode user,

I want to change my password while signed in,

so that I can maintain the security of my account.

---

### US-2 — Recover Forgotten Password

As a Hosted Mode user,

I want to recover my account when I forget my password,

so that I can regain access without administrator assistance.

---

### US-3 — Local Mode Isolation

As a Local Mode user,

I should never see hosted password management functionality,

because Local Mode does not use hosted authentication.

---

# Functional Requirements

## FR-1 Hosted Mode Availability

Password management functionality shall only be available when Hosted Authentication is enabled.

Local Mode shall not expose any password management controls or recovery workflows.

---

## FR-2 Change Password Entry Point

Authenticated users shall be able to initiate a password change from the Settings page.

---

## FR-3 Change Password Workflow

The Change Password workflow shall present the UI defined in the associated design artifact.

The workflow shall require:

- Current Password
- New Password
- Confirm New Password

The workflow shall provide actions to update the password or cancel the operation.

---

## FR-4 Change Password Validation

Before submitting a password change, Alice shall verify:

- All required fields are provided.
- The current password is authenticated by the Hosted Authentication provider.
- The new password complies with Alice's password policy.
- The new password and confirmation password match.

Validation errors shall be presented inline.

---

## FR-5 Password Change Completion

Upon successful validation, Alice shall submit the password change through the Hosted Authentication provider.

On success:

- Display a success notification.
- Close the overlay.
- Preserve or invalidate the current authenticated session according to the Hosted Authentication provider's behavior.

---

## FR-6 Forgot Password Entry Point

The Welcome screen shall provide a **Forgot Password?** action.

Selecting this action shall open the password recovery workflow.

This functionality shall only be available in Hosted Mode.

---

## FR-7 Password Recovery Request

The password recovery workflow shall present the UI defined in the associated design artifact.

The workflow shall:

- collect the user's email address
- validate email format
- submit a password recovery request through the Hosted Authentication provider

---

## FR-8 Recovery Request Confirmation

After a recovery request is submitted, Alice shall display a generic confirmation message indicating that recovery instructions have been sent if an account exists.

The confirmation shall not disclose whether the submitted email address is registered.

---

## FR-9 Recovery Session Detection

When Alice is launched through a valid password recovery link generated by the Hosted Authentication provider, the application shall detect the recovery session and present the Reset Password workflow instead of the standard login flow.

---

## FR-10 Reset Password Workflow

The Reset Password workflow shall present the UI defined in the associated design artifact.

The workflow shall require:

- New Password
- Confirm New Password

The workflow shall provide actions to update the password or cancel the operation.

---

## FR-11 Reset Password Validation

Before submitting the password reset, Alice shall verify:

- All required fields are provided.
- The new password complies with Alice's password policy.
- The new password and confirmation password match.
- The recovery session remains valid.

Validation errors shall be presented inline.

---

## FR-12 Password Reset Completion

Upon successful validation, Alice shall submit the new password through the Hosted Authentication provider.

On success:

- Display a success notification.
- Clear the recovery state.
- Return the user to the standard Welcome/Login flow.

---

## FR-13 Error Handling

Password management workflows shall gracefully handle authentication, network and recovery errors.

Sensitive implementation details shall never be exposed to users.

---

# Acceptance Criteria

## AC-1

Hosted users can initiate Change Password from Settings.

---

## AC-2

Change Password requires the current password, new password and confirmation password.

---

## AC-3

Forgot Password is available from the Welcome screen in Hosted Mode.

---

## AC-4

Submitting a recovery request sends a password recovery request through the Hosted Authentication provider.

---

## AC-5

Recovery request confirmation does not disclose whether the submitted email address exists.

---

## AC-6

Opening a valid recovery link automatically presents the Reset Password workflow.

---

## AC-7

Reset Password requires a new password and confirmation password.

---

## AC-8

Both Change Password and Reset Password enforce Alice's password policy.

---

## AC-9

Successful password changes and password resets notify the user and complete successfully.

---

## AC-10

No password management functionality is available in Local Mode.

---

# UX Considerations

- All password management interfaces shall be implemented as overlays in accordance with the associated design artifacts.
- Buttons shall provide loading states during authentication operations.
- Multiple submissions shall be prevented while requests are in progress.
- Validation shall be immediate where appropriate and presented inline.
- Focus management and keyboard accessibility shall be maintained throughout all workflows.

---

# Security Considerations

- Passwords shall never be logged or persisted outside the Hosted Authentication provider.
- Recovery requests shall never reveal whether an account exists.
- Password reset shall only be permitted during a valid recovery session.
- Alice shall rely on the Hosted Authentication provider for authentication and recovery validation.
- Alice shall not implement custom password recovery tokens.

---

---

# Notes

This feature is exclusive to Hosted Mode.

The Hosted Authentication provider remains the source of truth for authentication, password validation, and recovery. Alice is responsible for presenting a consistent user experience while delegating credential management operations to the authentication provider.
