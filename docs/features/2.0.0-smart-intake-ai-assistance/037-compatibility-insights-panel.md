# Feature Brief: 037 - Compatibility Insights Panel

## Summary

Introduce a redesigned Compatibility Insights Panel that combines deterministic compatibility scoring, visual compatibility indicators, skill proficiency mapping, and AI-generated compatibility analysis into a single cohesive experience.

This feature replaces the existing compatibility bar and compatibility notes implementation with a dedicated Compatibility module that helps users understand both their overall alignment to a role and the reasons behind that assessment.

The Compatibility Insights Panel consumes compatibility results generated by Feature 036 and presents them through a richer, more explainable user experience.

---

## Goals

- Make compatibility scores easier to understand
- Improve transparency of compatibility calculations
- Explain strengths and gaps in practical language
- Surface missing and developing skills visually
- Separate deterministic scoring from AI-generated analysis
- Give users control over AI generation and refresh behavior
- Maintain compatibility functionality even when AI services are unavailable

---

## Non-Goals

- Career coaching
- Resume rewriting
- Salary advice
- Interview preparation
- Conversational AI assistance
- User-editable compatibility scores
- User-editable AI-generated analysis
- Changes to the underlying compatibility scoring algorithm

---

## Compatibility Philosophy

Compatibility is intended to provide directional guidance, not application advice.

Scores indicate broad alignment between a user's profile and a role, but do not determine whether a user should apply. The Compatibility module should help users understand strengths, gaps, and opportunities without discouraging applications solely because of a low score.

The deterministic score remains the authoritative measure of compatibility. AI-generated insights exist to explain the score, not replace it.

---

## Feature Scope

This feature owns the entire Compatibility module experience, including:

- Compatibility module container
- Expanded and collapsed states
- Score visualization
- Compatibility verdict presentation
- Skill proficiency indicators
- AI-generated compatibility analysis
- Notes generation workflows
- Notes freshness management
- Empty states
- Error handling
- Responsive behavior

This feature consumes compatibility results generated by Feature 036 and does not modify the underlying scoring engine.

---

## Functional Requirements

### Compatibility Module

A dedicated Compatibility module shall be displayed within the Application Edit Modal.

The module shall:

- replace the existing compatibility bar
- replace the existing compatibility notes textarea
- support expanded and collapsed views
- be collapsed by default
- remain available in both Create and Edit modes

---

### Compatibility Score Presentation

The module shall display:

- compatibility score
- compatibility verdict
- compatibility summary

The compatibility score shall remain read-only.

The score shall always originate from the deterministic compatibility engine.

Users shall not be able to manually modify compatibility scores.

---

### Compatibility Tiers

The module shall support four compatibility tiers:

| Tier | Score Range |
|--------|--------|
| Low | 0 - 39 |
| Medium | 40 - 64 |
| High | 65 - 84 |
| Great | 85 - 100 |

Each tier shall have a distinct visual presentation.

---

### Collapsed View

The collapsed state shall provide a compact overview.

At minimum it shall display:

- compatibility score
- compatibility verdict
- compatibility summary or status

The compatibility score must remain visible while collapsed.

---

### Expanded View

The expanded state shall expose:

- score visualization
- compatibility verdict
- compatibility summary
- skill proficiency indicators
- AI-generated compatibility analysis
- generation status
- freshness status

---

### Skill Proficiency Mapping

Required Skills and Preferred Skills shall be rendered using profile-aware proficiency indicators.

Each skill shall resolve into one of three states:

| State | Definition |
|---------|---------|
| Proficient | Skill exists on profile with rating >= 3 |
| Learning | Skill exists on profile with rating < 3 |
| Missing | Skill does not exist on profile |

Both Required Skills and Preferred Skills shall use the same proficiency rendering model.

A legend shall be provided to explain proficiency indicators.

---

### AI Analysis

The system shall support AI-generated compatibility analysis.

Analysis shall explain:

- notable strengths
- notable gaps
- alignment observations
- relevant compatibility context

Analysis should remain concise, practical, and grounded in available profile and job data.

---

### Notes Lifecycle

Compatibility analysis shall support the following states:

- none
- generating
- fresh
- stale
- error

Each state shall have a dedicated UI presentation.

---

### Notes Generation

If no analysis exists:

- compatibility score remains visible
- users may generate notes manually

Notes shall not be generated automatically.

---

### Notes Refresh

The compatibility score shall update automatically whenever underlying compatibility data changes.

AI-generated notes shall not refresh automatically.

When profile or application data changes after note generation:

- notes become stale
- score remains current
- users may manually refresh notes

---

### Stale Notes

The module shall clearly indicate when notes are stale.

The stale state shall communicate:

- why notes are stale
- that the score remains current
- that notes may be refreshed manually

---

### Regeneration

Users shall be able to:

- generate notes
- regenerate notes
- refresh stale notes

Each operation shall produce a newly generated analysis.

---

### Failure Handling

Compatibility scoring and AI analysis shall operate independently.

If AI generation fails:

- compatibility scores remain available
- skill indicators remain available
- verdicts remain available
- users may retry generation

AI failures shall never block application management workflows.

---

### Empty States

The module shall support dedicated empty states for:

#### No Profile

Displayed when insufficient profile information exists to calculate compatibility.

The user shall be encouraged to complete their profile.

#### No Analysis

Displayed when compatibility scoring exists but notes have not yet been generated.

The user shall be able to generate notes.

#### Generation Failure

Displayed when note generation fails.

The user shall be able to retry.

---

### Responsive Behavior

The Compatibility module shall support:

- desktop layouts
- tablet layouts
- mobile layouts
- bottom-sheet modal layouts

All functionality must remain available across supported viewport sizes.

---

## Technical Notes

### Architecture

Compatibility scoring remains owned by Feature 036.

Feature 037 consumes compatibility results and renders them within the UI.

The Compatibility module shall not contain scoring logic.

---

### AI Usage Philosophy

AI is responsible only for generating explanations.

The compatibility score itself remains deterministic and explainable.

AI output supplements the score and does not replace it.

---

### Cost Management

Analysis generation should be intentionally user-driven.

Automatic regeneration shall be avoided to reduce unnecessary AI usage and operating costs.

---

### Prompting Guidelines

Prompts should encourage:

- concise explanations
- practical observations
- grounded conclusions
- minimal hallucination
- actionable gap identification

Prompts should avoid:

- career advice
- hiring predictions
- application recommendations
- speculative claims

---

## Edge Cases

- Empty profile
- Sparse profile data
- Very short job descriptions
- Missing required skills
- Large skill lists
- Contradictory extracted data
- Failed AI requests
- Stale notes after profile updates
- Stale notes after application updates
- Low compatibility scores with strong experience signals
- High compatibility scores with sparse profile data

---

## Acceptance Criteria

- Compatibility section is collapsed by default
- Compatibility score remains visible when collapsed
- Expanded view exposes score, skills, and analysis
- Compatibility score remains read-only
- Skill proficiency indicators render correctly
- Notes are not generated automatically
- Users may generate notes manually
- Users may regenerate notes manually
- Users may refresh stale notes manually
- Profile changes can make notes stale
- Application changes can make notes stale
- Compatibility score updates independently of notes
- AI failures do not block compatibility scoring
- Empty states render correctly
- Responsive layouts function correctly on mobile and desktop
